[SYSTEM: E-MUNICIPALITY WEB PORTAL]
The E-Municipality portal runs on a server with IP address 10.10.1.50 behind Nginx.
PostgreSQL is used as the database.
When an Nginx 502 Bad Gateway error occurs, first check /var/log/nginx/error.log, then run the command "systemctl restart e-belediye-backend".

[SYSTEM: CITY INFORMATION SYSTEM (KBS) & GEOGRAPHIC INFORMATION SYSTEM (GIS)]
KBS and GIS services run in the Kubernetes environment under the "kbs-prod" namespace.
If MapServer pods experience a memory leak, restart them safely using the command "kubectl rollout restart deployment/mapserver -n kbs-prod".

[SYSTEM: MUNICIPALITY BACKUP AND DATA CENTER]
The automated backup script running at 02:00 AM daily writes to the /mnt/belediye_backup directory via RSYNC.
If disk usage exceeds 90%, Prometheus triggers the "BelediyeDiskFull" alert. In an emergency, the /tmp/old_logs folder can be deleted.

[SYSTEM: MUNICIPALITY E-SIGNATURE AND BELGENET (EDMS)]
The Electronic Document Management System (EDMS / EBYS) operates on a Java/Tomcat server with IP address 10.10.1.60.
When an E-Signature verification error occurs, access to Kamu SM OCSP servers via ports 8080 and 8443 should be checked.
If the Tomcat service freezes, run the command "systemctl restart tomcat9".

[SYSTEM: SMART CITY & SENSOR NETWORKS (IoT)]
The IoT Gateway service, which collects traffic, air quality, and Smart Stop data, runs as a Docker container named "iot-gateway".
If data flow is interrupted, inspect the MQTT Broker (Mosquitto) connection and refresh the service using the command "docker restart iot-gateway".

[SYSTEM: DIGITAL REGISTRY AND COLLECTION DATABASE]
The citizen registry and property/environmental tax database resides on a PostgreSQL Cluster with IP address 10.10.2.10.
If a "Too many connections" error is encountered in the database, reset the PgBouncer connection pooler service using "systemctl restart pgbouncer".

[SYSTEM: MUNICIPALITY NETWORK & FIREWALL]
E-government integration ports are defined on the external municipal firewall.
When system administrators connect externally via VPN, they must ensure port 1194 UDP is open on the FortiClient / OpenVPN server.
